# ?? About Me:
My skills are making things work, not knowing a billion facts. If I need to fix a system I'll identify the problems, check the logs, and look up the errors. If I need to implement a solution I'll research the right solution, implement and document it, later on only really have a general idea of how it works unless I interact with it frequently. It's why it's documented. This means that even if I don't know how to do something, I have no choice but to learn within a day or two or get it done.
